BROWSER PERMISSION FIX (if still failing)
═══════════════════════════════════════════════════════════════════════════════

Chrome/Edge:
  1. Click 🔒 Lock icon in address bar
  2. Find "Location" 
  3. Change to "Allow"
  4. Refresh page (F5)

Firefox:
  1. Go to about:preferences#privacy
  2. Find "Location" permissions
  3. Find localhost:3000 and set to "Allow"
  4. Refresh page (F5)

Safari:
  1. Settings → Privacy → Location
  2. Find localhost and set to "Allow"
  3. Refresh page (F5)

FALLBACK OPTIONS
═══════════════════════════════════════════════════════════════════════════════

If geolocation fails for any reason, users can still:

1. ✅ Drag the blue marker on the map to their location
2. ✅ Click anywhere on the map to place the marker
3. ✅ Manually verify and adjust coordinates

PRODUCTION DEPLOYMENT NOTE
═══════════════════════════════════════════════════════════════════════════════

HTTPS REQUIRED:
  • Geolocation API requires HTTPS on production
  • localhost:3000 works without HTTPS for testing
  • Vercel auto-provides HTTPS (✅ ready to deploy)
  • Self-hosted: Use Let's Encrypt (free HTTPS)

If using self-hosted and HTTP:
  → Geolocation will fail
  → Users must manually select location on map (still works!)
  → Consider upgrading to HTTPS

QUICK TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "Not supported" message
  Solution: Use Chrome, Firefox, Safari, or Edge (not old IE)

Problem: Permission denied
  Solution: Grant location permission in browser settings

Problem: Button says "Detecting..." forever
  Solution: Wait 15 seconds (timeout), check internet connection

Problem: No alert appears
  Solution: Open DevTools (F12) → Console tab to see errors

Problem: Coordinates show but map doesn't update
  Solution: Refresh page (F5)

For all issues:
  → See GEOLOCATION_FIX.md for comprehensive troubleshooting
